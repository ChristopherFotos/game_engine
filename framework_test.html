So what's going on here?

Props.js is a JavaScript library that lets designers work with physics-based motion -- without having to know anything about physics. All the motion that you're seeing on screen is governed by velocity and acceleration vectors. The library also includes algorithms for detecting and handling collision. You can see it in action in the demo above -- see how each block moves realistically in response to collision? that's a product Props.js' collision handling algorithm. The API also lets you sub in your own collision handling and detecting algorithms.

Props.js API makes it easy to start an animated scene. It only takes two lines of JavaScript:

const scene = new Scene("canvas", window.innerWidth, window.innerHeight); startScene(scene);

first line creats a scene and saves it to a constant called 'scene.' It takes three arguments: the ID of the HTML5 canvas you want to target, and your desired width and height.

the second line starts a scene. It only takes one argument, which is the scene you want to start (in this case our scene is simply named 'scene').

Props.js also provides a simple syntax for adding animated objects (or Props) to your scene. Simple call the Scene.add() method, specify the scene you want to add the object to, as the first argument. As the second argument, pass in an object full of options describing what you want the object to look like, and how you want it to behave:

scene.add(
scene,
shape: 'rectangle',
width: 100
height: 100,
x: 300
y: 500
strokeColor: 'gray',
fill: true,
fillColor: green
accelMag: 0,
speed: 15,
solid: true,
direction: 5
}
)

In this example, we're adding to the scene we set up in the previous code snippet. remember -- you call the method on the scene you're adding to (in our case, our scene is named 'scene'), and you pass the name of that seen into the method as the first argument.

The second argument is an object containing options that describe the shape and behaviour of our prop.
